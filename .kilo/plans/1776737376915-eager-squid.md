# Sound Effects Implementation Plan

## Current State
- Scanned all `58` `.wav` clips under `Assets/Sound/Sound Effects`.
- GUID search across `*.unity`, `*.prefab`, `*.asset`, `*.controller`, `*.mat`, `*.anim`, and `*.cs` found only `1` referenced clip: `Assets/Sound/Sound Effects/Overworld Misc/Treasure Chest Opening.wav`, wired in `Assets/Prefabs/Interactables/TreasureChest.prefab` through `Assets/Scripts/Interactions/TreasureChest.cs`.
- `57` clips currently appear unused.
- Existing audio architecture is music-only: `Assets/Scripts/Audio/TempoMusicDirector.cs`, `TempoMusicSet.cs`, and `TempoMusicZone.cs`.
- Existing one-shot SFX pattern is minimal: `Assets/Scripts/Interactions/TreasureChest.cs` uses `AudioSource.PlayClipAtPoint`.
- Confirmed first-pass direction from user:
- Use a minimal reusable one-shot helper rather than a central SFX manager.
- Use `Jorge/BreadHealingCRUNCH (1).wav` for sweetbread healing.
- Trigger enemy damaged sounds from current knockback hooks.
- Defer edge-case clips for now: guitar loops, `Explosion.wav`, and `Introduction Scene Rumble.wav`.
- Pista dialogue should have voice on non-sign dialogue, while Pista yaps should be reserved for Pista movement/travel rather than dialogue typewriter blips.
- Treat `Pista Dialogue 1-6.wav` as a shared random voice bank for non-sign dialogue.
- Use `Title Menu Button Press Start.wav` for confirm only, not menu navigation.
- Map `smallAttack*` and `ShortSkeletonDamaged` to melee skeletons, and `tallAttack*` and `TallSkeletonDamaged` to ranged skeletons.
- Play Jorge hit sounds only on non-lethal player damage.

## Script Additions Needed

### Recommended Shared Audio Additions
- Add a lightweight reusable one-shot SFX helper under `Assets/Scripts/Audio/`.
- Recommended shape: local `AudioSource`, optional mixer-group support, `Play(AudioClip)` and `PlayRandom(AudioClip[])`, optional pitch/volume variance.
- This keeps changes in existing gameplay scripts small and avoids introducing a large global audio manager before it is needed.

## Implementation Notes
- Minimal recommended first pass:
  - Build a reusable one-shot SFX helper.
  - Implement clear one-shot interactions first: menu confirm, lantern swing, hit/death, pickups, torch lit, door open, crate break, switch toggle, puzzle solved/failed, radio switch, enemy spawn/attack, time-up.
  - Implement non-sign Pista dialogue voice once the assignment model is confirmed.
  - Implement Pista yaps on send/return/travel transitions.
  - Defer guitar loops, `Explosion.wav`, and `Introduction Scene Rumble.wav`.
- `Treasure Chest Opening.wav` should be left alone initially because it is already hooked up and working.
- Several existing UnityEvents can reduce code churn once the shared one-shot helper exists, but some core interaction points still need direct code changes because they currently have no event at the exact moment the SFX should fire.
